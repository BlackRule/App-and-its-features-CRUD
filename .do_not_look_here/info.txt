Пойдет ли такой стек? Начинаю делать так:  0)PostgreSQL 1) https://gin-gonic.com/ web framework (GO) в том числе для статики + https://gqlgen.com/ graphql generator. 2) React build (статич файлы). 3) Всё это в Docker
Так-же хочу своровать (распарсить) данные (фичи приложений таких как Zoom такие как Remote Control ) с сайтов-аналогов моего (alternativeto.net например) чтоб наполнить мой.
Позже 2 заменить можно на NextJs'ом сгенеренные статические файлы. (SSR).
Так-же wundergraph существует который почему-то пропагандирует что лучше не выставлять наружу graphQL с сервера (и не исп-ть GQ клиентскую библиотеку) а исп-ть wundergraph который на основе graphql схемы выставит наружу REST (jsonRPC) и сгенерирует клиентский код обращающийся к нему. Якобы так с клиента снимается нагрузка по парсингу graphql а кэширование происходит на уровне HTTP (и CDN'ов ч/з которые запрос проходит и тд).
